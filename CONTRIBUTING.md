# Contributing to WebSec Scanner

Thanks for your interest in contributing! This project is small and focused — contributions that improve usefulness, reliability, or safety are welcome.

## How to contribute

1. Fork the repository and create a feature branch from `main`:

```bash
git checkout -b fix/some-bug
```

2. Make your changes in a clear, focused commit. Keep commits small and descriptive.

3. Run the project locally and verify your changes (see `README.md` for setup).

4. Push your branch and open a pull request describing the change and why it's needed.

## Issues

- Open an issue for bugs, feature requests, or questions.
- Please provide clear reproduction steps for bugs and sample commands/output when possible.

## Coding style

- Keep code simple and readable.
- Use descriptive variable and function names.
- Follow existing project style (PEP8 for Python).

## Tests

- There are no automated tests at present. For larger changes, include a small script or example demonstrating the change.

## Security & responsible disclosure

- Do not include exploit code or tests that could be destructive.
- If you discover a security issue, please open a private issue or contact the repository owner instead of posting details publicly.

## License

By contributing, you agree that your contributions will be licensed under the project's license (add license in `LICENSE` file).

Thanks — we appreciate your help!
